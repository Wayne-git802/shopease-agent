# 安全政策

## 报告安全漏洞

**请不要**通过 GitHub Issues 报告安全漏洞。

如果你发现了安全漏洞，请通过以下方式私密地向我们报告：

1. **GitHub 私密报告**（推荐）
   - 访问仓库的 Security 标签
   - 点击 "Report a vulnerability"
   - 填写详细信息

2. **邮件联系**
   - 发送邮件到项目维护者
   - 主题行包含 "[SECURITY]"
   - 描述漏洞的详细信息

## 信息应包含

请在报告中包含以下信息：

- 漏洞的描述
- 受影响的版本
- 复现步骤（如果可能）
- 潜在的影响
- 建议的修复方案（如果有）

## 响应时间

我们承诺：

- 在 48 小时内确认收到报告
- 在 7 天内提供初步评估
- 在 30 天内提供修复方案

## 致谢

我们会在修复后的发布说明中感谢安全研究人员，除非他们要求保持匿名。

## 安全最佳实践

在使用本项目时，请遵循以下最佳实践：

1. **定期更新依赖**
   ```bash
   pip install --upgrade -r requirements.txt
   npm update
   ```

2. **不要在代码中存储密钥**
   - 使用 `.env` 文件（不提交到 Git）
   - 使用环境变量
   - 使用密钥管理服务

3. **验证输入**
   - 对所有用户输入进行验证
   - 使用白名单而非黑名单

4. **保持日志安全**
   - 不记录敏感信息
   - 定期轮换日志

5. **使用 HTTPS**
   - 所有网络通信都应使用 HTTPS

## 支持的版本

我们仅为最新版本提供安全支持。请始终保持更新。
